description: "How DPNP selects SYCL devices using compute-follows-data, how to create arrays on a specific device, and how to inspect device properties with dpctl."

DPNP follows a *compute follows data* paradigm: every array is tied to a SYCL device and a SYCL queue, and operations on that array run on the same device automatically. By default DPNP selects the best available device — typically a GPU — but you can override that choice per-array with the `device=` parameter or per-queue with the `sycl_queue=` parameter.

## Default device selection

When you create an array without specifying a device, DPNP delegates device selection to dpctl. dpctl's default selector prefers a GPU when one is available and falls back to a CPU otherwise.

```python
import dpnp as np

# Array lands on the default device (GPU if available, CPU otherwise)
x = np.asarray([1, 2, 3])
print("Array x allocated on the device:", x.device)
# e.g. Device(level_zero:gpu:0)

y = np.sum(x)
print("Result y is located on the device:", y.device)  # same device as x
print("y =", y)  # 6
```

<Note>
The environment variable `SYCL_QUEUE_THREAD_POOL_SIZE` is set to `"6"` by default when you import `dpnp`. You can override it before importing dpnp to tune thread-pool size for your workload.
</Note>

## Selecting a specific device

Pass a filter selector string or a `dpctl.SyclDevice` object to the `device=` parameter of any array constructor to target a specific device. Available selector strings include `"gpu"`, `"cpu"`, `"level_zero:gpu:0"`, and `"opencl:cpu:0"`.

```python
import dpctl
import dpnp

# Use dpctl to select CPU and GPU devices explicitly
cpu_device = dpctl.select_cpu_device()
a = dpnp.ones((1024,), device=cpu_device)
print(a.device)  # Device(opencl:cpu:0) or similar

b = dpnp.linspace(0, dpnp.pi, num=10**6, device="gpu")
f = 1 + b * dpnp.sin(b)
print(b.device)  # Device(level_zero:gpu:0)
```

<Tip>
Use `dpctl.select_cpu_device()`, `dpctl.select_gpu_device()`, or `dpctl.select_default_device()` from the `dpctl` package to get a `SyclDevice` object you can pass as `device=`.
</Tip>

## Creating arrays on specific devices

Every array-creation routine in DPNP accepts a `device=` keyword argument. You can pass a filter selector string, a `dpctl.SyclDevice` object, a `dpctl.SyclQueue`, or a `dpnp.tensor.Device` returned by `array.device`.

```python
import dpctl
import dpnp

# Pass a filter selector string
x = dpnp.array([1, 2, 3], device="gpu")

# Pass a dpctl.SyclDevice
cpu_device = dpctl.select_cpu_device()
y = dpnp.array([1, 2, 3], device=cpu_device)
print(y.device)  # Device(opencl:cpu:0) or similar

# Use the device of an existing array to keep data co-located
z = dpnp.zeros_like(x)          # same device as x
w = dpnp.ones((5,), device=x.device)  # same device as x
```

<CodeGroup>

```python zeros / ones / empty
import dpnp

# All creation functions accept device=

c = dpnp.empty((4, 4), device="gpu")
d = dpnp.full((4, 4), fill_value=3.14, device="gpu")
```

```python arange / linspace
import dpnp

r = dpnp.arange(100, device="gpu")
l = dpnp.linspace(0, 1, num=1000, device="gpu")
```

### The `usm_type` parameter

In addition to `device=`, creation functions accept `usm_type=` to control the type of Unified Shared Memory (USM) allocation:

| `usm_type` | Description |
|---|---|
| `"device"` (default) | Allocated on device memory; fastest for GPU kernels. |
| `"shared"` | Accessible from both host and device; may have migration overhead. |
| `"host"` | Allocated in host memory but accessible by the device. |

```python
import dpnp

# Shared USM — accessible from Python and from device kernels
x = dpnp.ones((1024,), usm_type="shared", device="gpu")
print(x.usm_type)  # shared
```

## Checking device properties

Every `dpnp.ndarray` exposes two device-related attributes.

### `.device` — Array API device object

Returns a `dpnp.tensor.Device` object containing the `SyclQueue` associated with the array. This is the Array API standard notion of a device.

```python
import dpnp as np

x = np.ones(10)
print(x.device)         # Device(level_zero:gpu:0)
print(type(x.device))   # <class 'dpnp.tensor._device.Device'>
```

### `.sycl_device` — underlying SYCL device

Returns the `dpctl.SyclDevice` object, which exposes detailed hardware information.

```python
import dpnp

x = dpnp.zeros((4,), device="gpu")
dev = x.sycl_device

print(dev.name)           # e.g. "Intel(R) UHD Graphics 770"
print(dev.driver_version) # e.g. "1.3.26032"
print(dev.has_aspect_fp64)  # True if float64 is natively supported
print(dev.max_compute_units)
```

<Warning>
Not all GPU devices support 64-bit floating point natively. Check `device.has_aspect_fp64` before running float64 workloads. The helper `dpctl.select_default_device()` used in `example1.py` demonstrates this pattern.
</Warning>

### `.sycl_queue` — the SYCL queue

Returns the `dpctl.SyclQueue` that was used to create the array.

```python
import dpnp

x = dpnp.arange(10, device="gpu")
q = x.sycl_queue
print(q.sycl_device.name)
```

## Device-aware array creation patterns

<CardGroup cols={2}>
  <Card title="Co-locate output with input" icon="link">
    Pass `device=input.device` or use `*_like` variants to keep the result on the same device without hardcoding a selector string.
  </Card>
  <Card title="Explicit queue threading" icon="cpu">
    Pass a `dpctl.SyclQueue` object as `sycl_queue=` for fine-grained control over in-order vs. out-of-order queues and SYCL properties.
  </Card>
  <Card title="GPU-first with CPU fallback" icon="server">
    Wrap `device="gpu"` in a `try/except` block and fall back to `device="cpu"` as shown in `example_cfd.py`.
  </Card>
  <Card title="Inspect before allocating" icon="magnifying-glass">
    Use `dpctl.select_default_device().has_aspect_fp64` before choosing float64 dtype to avoid silent precision loss.
  </Card>
</CardGroup>

```python
import dpnp as np

# GPU-first with CPU fallback (from example_cfd.py)
x = np.empty(3)
try:
    x = np.asarray([1, 2, 3], device="gpu")
except Exception:
    print("GPU device is not available, falling back to CPU")
    x = np.asarray([1, 2, 3], device="cpu")

print("Array x allocated on the device:", x.device)

y = np.sum(x)
print("Result y is located on the device:", y.device)
print("y =", y)  # 6
```
