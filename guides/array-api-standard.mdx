The [Python Array API standard](https://data-apis.org/array-api/latest/) is a community specification that defines a common interface for array libraries. Code written against the Array API standard can run on NumPy, CuPy, PyTorch, JAX, DPNP, and any other compliant library without modification. DPNP implements this standard so that your data-science and scientific-computing code can target Intel GPUs with zero library-specific changes.

## What the Python Array API standard is

The Array API standard specifies a minimal set of functions, dtypes, and array attributes that every compliant library must provide. It covers:

- Array creation routines (`zeros`, `ones`, `arange`, `linspace`, …)
- Mathematical functions (`sin`, `cos`, `exp`, `log`, …)
- Linear algebra (`matmul`, `tensordot`, `linalg.solve`, …)
- FFT (`fft.fft`, `fft.ifft`, …)
- Data type promotion rules
- Device and array introspection via `__array_namespace__` and `__array_namespace_info__`

When a function accepts any array-like input, it can use `x.__array_namespace__()` to obtain the correct library namespace and call functions on it — without ever importing that library directly.

## DPNP compliance level

DPNP's main namespace, `dpnp.fft`, and `dpnp.linalg` are compatible with the **2024.12 version** of the Python Array API standard. DPNP also implements the [array API inspection utilities](https://data-apis.org/array-api/latest/API_specification/inspection.html) via `dpnp.__array_namespace_info__()`.

```python
import dpnp

# The API version string
print(dpnp.__array_api_version__)  # '2024.12'

# Inspection utilities
info = dpnp.__array_namespace_info__()
print(info.default_dtypes())
print(info.dtypes())
```

<Note>
`dpnp.__array_api_version__` is re-exported from `dpnp.tensor`. It reflects the version of the standard that DPNP's tensor layer was built against.
</Note>

## Using DPNP via `__array_namespace__`

Every `dpnp.ndarray` implements `__array_namespace__()`. Call it to retrieve the `dpnp` module as a namespace, then use it to call functions in a library-agnostic way.

```python
import dpnp

x = dpnp.ones((4, 4), device="gpu")

# Get the namespace from the array itself
xp = x.__array_namespace__()

# Now use `xp` instead of importing dpnp directly
y = xp.zeros_like(x)
z = xp.matmul(x, y + xp.eye(4))
print(type(z))          # <class 'dpnp.dpnp_array.dpnp_array'>
print(z.device)         # Device(level_zero:gpu:0)
```

You can also request a specific API version:

```python
import dpnp

x = dpnp.arange(10, device="gpu")
xp = x.__array_namespace__(api_version="2024.12")
```

## Writing array-agnostic code

The key pattern is: accept any array as input, retrieve its namespace, and use only that namespace for all operations. The function below works with NumPy, DPNP, or any other Array API-compliant library.

```python
def normalize_rows(x):
    """
    Normalize each row of a 2-D array to unit L2 norm.

    Works with any Array API-compliant library (NumPy, DPNP, CuPy, …).
    """
    xp = x.__array_namespace__()
    norms = xp.sqrt(xp.sum(x ** 2, axis=1, keepdims=True))
    return x / norms
```

Call it with DPNP:

```python
import dpnp

data = dpnp.array([[3.0, 4.0], [1.0, 0.0]], device="gpu")
result = normalize_rows(data)
print(result.device)   # Device(level_zero:gpu:0)
```

Call the same function with NumPy — no changes required:

```python
import numpy as np

data_cpu = np.array([[3.0, 4.0], [1.0, 0.0]])
result_cpu = normalize_rows(data_cpu)
```

### Checking the namespace at runtime

If you need to branch on the library (for example to use a library-specific optimization), compare the namespace object rather than the array type:

```python
import numpy as np
import dpnp

def compute(x):
    xp = x.__array_namespace__()

    if xp is dpnp:
        # DPNP-specific path — e.g., pass sycl_queue explicitly
        result = xp.fft.fft(x)
    else:
        result = xp.fft.fft(x)

    return result
```

## Inspection utilities

`dpnp.__array_namespace_info__()` returns a namespace containing functions that describe DPNP's capabilities. This is part of the Array API standard inspection API.

```python
import dpnp

info = dpnp.__array_namespace_info__()

# Default floating-point and integer dtypes
defaults = info.default_dtypes()
print(defaults)
# e.g. {'real floating': dpnp.float64, 'complex floating': dpnp.complex128, ...}

# All supported dtypes (optionally filtered by kind)
all_dtypes = info.dtypes()
float_dtypes = info.dtypes(kind="real floating")
```

<Tip>
Use `info.default_dtypes()` to write dtype-agnostic code that respects the platform's preferred precision rather than hardcoding `float64` or `float32`.
</Tip>

## DLDevice type enumeration

DPNP re-exports `DLDeviceType` from its tensor layer. DLDevice types are used in DLPack data exchange (see [Interoperability](/guides/interoperability)) and in the `__dlpack_device__` method.

```python
import dpnp

# DLDeviceType values
print(dpnp.DLDeviceType)
# Includes kDLCPU, kDLCUDA, kDLOpenCL, kDLOneAPI, etc.

x = dpnp.zeros(4, device="gpu")
device_type, device_id = x.__dlpack_device__()
print(device_type, device_id)  # e.g. DLDeviceType.kDLOneAPI 0
```
